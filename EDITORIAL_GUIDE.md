# EDITORIAL_GUIDE.md

# Editorial Guide

This guide defines the editorial standards for all written material within the Life School repository.

Its purpose is to help every contributor write clearly, consistently, and faithfully to the Constitution.

It applies equally to human contributors and artificial intelligence systems.

---

# The Editorial Purpose

Writing exists to serve understanding.

Every document should make Life School easier to understand.

If writing makes understanding more difficult, it should be improved.

---

# The Constitution Comes First

Every document derives from the Constitution.

Editorial improvements may clarify the Constitution.

They shall not redefine it.

If uncertainty exists, Constitutional Anchor CA-001 prevails.

---

# One Document, One Purpose

Every document should answer one primary question.

Avoid combining unrelated subjects into the same document.

If a document begins serving multiple independent purposes, consider dividing it.

If a document remains clear as one document, keep it together.

---

# Write for Understanding

Prefer writing that is:

- clear,
- precise,
- direct,
- understandable,
- timeless.

Avoid writing that is unnecessarily:

- impressive,
- complicated,
- abstract,
- repetitive.

The objective is understanding, not admiration.

---

# Prefer Simple Language

Use language that an interested reader can understand without specialized knowledge.

When technical language is necessary, explain it.

Simple writing is not simplistic writing.

It is respectful writing.

---

# Every Sentence Must Earn Its Place

Before keeping a sentence, ask:

- Does it improve understanding?
- Does it support the document's purpose?
- Can it be written more clearly?
- Can it be removed without losing meaning?

If a sentence serves no purpose, remove it.

---

# Derive, Don't Invent

New ideas should emerge from the Constitution.

Avoid introducing principles that cannot be traced back to the Foundation.

Every significant addition should answer:

- What does this derive from?
- Why is it necessary?

---

# Prefer Clarity Over Volume

Longer documents are not better documents.

Additional words should only exist when they increase understanding.

When two versions communicate the same meaning, prefer the clearer one.

---

# Distinguish Facts From Interpretation

When writing:

- identify established facts,
- distinguish observations,
- distinguish interpretations,
- acknowledge uncertainty honestly.

Readers should always understand which is which.

---

# Avoid False Certainty

Do not present assumptions as facts.

Do not claim certainty where uncertainty exists.

Admitting uncertainty strengthens credibility.

---

# Continuous Improvement

Nothing is ever complete.

Every document may be improved through honest examination.

Improvement should strengthen understanding rather than introduce unnecessary complexity.

---

# Respect the Reader

Assume the reader is thoughtful.

Do not manipulate.

Do not exaggerate.

Do not write to impress.

Write to help the reader understand.

---

# Preserve Timelessness

Avoid references that unnecessarily tie a document to:

- current technology,
- temporary trends,
- specific software,
- contemporary politics,
- short-lived terminology.

Write so the document remains understandable decades from now.

---

# Consistency

Use terminology consistently throughout the repository.

If a constitutional term already exists, continue using it.

Do not create multiple names for the same concept.

When terminology changes, update the Constitution before updating derived documents.

---

# FP-001 Usage Discipline

Use FP-001 consistently and without overuse.

When writing, choose one of these levels intentionally:

- Full wording: use when defining the principle in foundational or public trust statements.
- FP-001 reference: use when documenting traceability across governance, decisions, revisions, and institutional learning.
- Indirect reflection: use when operational text already communicates humility, reviewability, and correction.
- No explicit reference: use in technical details where repeating the principle does not improve understanding.

The goal is coherence, not repetition.

---

# Compression Before Expansion

Whenever editing a document, ask:

- Can this be shorter?
- Can this be clearer?
- Can repetition be removed?
- Can structure be improved?

Only expand a document when doing so meaningfully improves understanding.

---

# Editorial Responsibility

Every contributor shares responsibility for maintaining the quality of Life School's written work.

Good editing is considered a contribution equal in value to writing new content.

Improving clarity strengthens the institution.

---

# Closing Principle

Write every document as though it may still be read one hundred years from now.

Write every sentence as though it may become part of someone's lifelong learning.

Write faithfully.

Write clearly.

Write with humility.

Always serve the Purpose.
