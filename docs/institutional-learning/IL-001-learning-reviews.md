# IL-001
# Learning Reviews

Version: Foundation Edition v0.0.1 (Draft)

Status: Operational Draft

---

# Purpose

Learning Reviews examine important institutional changes or discoveries.

---

# Trigger Conditions

Run a Learning Review when:

- a constitutional interpretation shifts,
- a major product behavior changes,
- a governance or transparency policy changes,
- a significant mistake is discovered,
- uncertainty accumulates around an active position.

---

# Required Fields

Each Learning Review should include:

- Review ID,
- Date,
- Trigger,
- What changed,
- Why it changed,
- What was learned,
- Mistakes identified,
- Remaining uncertainty,
- How understanding deepened,
- Related artifacts,
- Next review condition.

---

# Output

The canonical output is one LR entry using templates/LR-TEMPLATE.md.

---

# Logging

Each completed Learning Review should be logged in registers/learning-review-log.md.
